# 🎵 MelodyCraft 可行性分析报告

> 日期：2025-05-25 | 版本 V1.0

---

## 一、项目概述

MelodyCraft 是面向普通音乐爱好者的可视化音乐创作工具。用户通过拖拽乐器、在钢琴卷帘上编排音符、结合 AI 辅助和歌词编辑，可以轻松完成一首完整的音乐作品。

### 核心目标

> 让没有乐理基础的人，也能在 10 分钟内做出一段属于自己的旋律。

### 用户画像

- 普通音乐爱好者，非专业制作人
- 可能学过一点乐器但不会用 DAW
- 想创作但被专业软件门槛劝退
- 年龄 16-35 岁，习惯网页端操作

---

## 二、市场分析

### 竞品格局

| 产品 | 定位 | 优点 | 缺点 |
|------|------|------|------|
| **GarageBand** | 免费 DAW | 触控优化好、音色丰富 | 仅限 Apple 生态 |
| **BandLab** | 社交+创作 | 免费、社交属性强 | 编辑器体验一般 |
| **Chrome Music Lab** | 音乐教育 | 极简、无需注册 | 功能太少 |
| **FL Studio** | 专业 DAW | 功能强大 | 价格高、学习曲线陡 |
| **Mubert / Boomy** | AI 生成 | 一键出歌 | 没有创作感 |

### 市场空白

市面上要么太专业（DAW），要么太简单（AI 一键生成）。**中间有一个明确的地带：让用户有"我在创作"的参与感，但不需要学乐理。** 这正是 MelodyCraft 的位置。

## 三、技术可行性

### 技术栈

| 技术 | 用途 | 评估 |
|------|------|------|
| Web Audio API | 音频播放/合成 | ✅ 所有现代浏览器支持 |
| Tone.js | 高级音频封装 | ✅ 成熟开源库 |
| Canvas / WebGL | 钢琴卷帘可视化 | ✅ Three.js 经验可复用 |
| IndexedDB | 本地存储项目 | ✅ 足够支撑 MVP |

### 风险评估

| 风险 | 等级 | 应对 |
|------|------|------|
| 音频延迟 | 🟡 中 | 前期以 Sample/LOOP 为主 |
| 浏览器兼容 | 🟢 低 | 聚焦 Chrome + Safari |
| 文件体积 | 🟡 中 | 使用 MIDI + 音色合成 |
| 移动端性能 | 🟡 中 | Canvas 渲染优化 |

✅ **技术上可行。** Web Audio API + Tone.js 生态已足够支撑 MVP。

## 四、MVP 范围

### 包含

- 钢琴卷帘编辑（核心）：网格化界面、点击添加/删除音符、播放
- 多音轨支持：至少 4 轨，不同乐器，静音/独奏
- 内置乐器音色：钢琴、吉他、贝斯、鼓组、弦乐
- 项目保存/打开：JSON 格式，本地导出导入
- 导出音频：导出为 WAV

### 不包含（后续迭代）

| 功能 | 版本 |
|------|------|
| AI 辅助生成和弦/旋律 | V1.1 |
| 歌词编辑 & 人声合成 | V1.2 |
| 社交分享 | V1.3 |
| 移动端 App | V2.0 |
