## 一、100 万用户意味着什么

### 数据估算

| 指标 | 估算值 |
|------|--------|
| MAU | 100 万 |
| DAU | 10-20 万 |
| 同时在线 | 2,000-5,000 |
| 每月存储 | ~10GB 项目数据 |
| 每月带宽 | 3-5TB |
| API 请求 | 50-100 req/s（峰值） |

## 二、架构演进路线

### 阶段 1：MVP（0 → 1 万用户）

**纯前端 + 静态托管，零成本**

```
用户浏览器 ──→ GitHub Pages / Cloudflare Pages
(SPA)              (静态资源 + CDN)
     │
     └── IndexedDB（全部本地存储）
```

| 组件 | 方案 |
|------|------|
| 前端 | HTML/CSS/JS |
| 存储 | IndexedDB |
| 托管 | GitHub Pages |
| CDN | Cloudflare（免费） |
| **月成本** | **¥0** |

---

### 阶段 2：增长期（1 万 → 10 万用户）

**前端 + 轻量后端 + 用户系统**

```
用户浏览器 ──→ Cloudflare CDN ──→ Vercel (API + SSR)
                                        │
                                   Supabase (DB + Auth)
```

| 组件 | 方案 |
|------|------|
| 前端框架 | React / Vue + Vite |
| 后端 | Vercel Edge Functions |
| 数据库 | Supabase（免费额度足够） |
| 存储 | Cloudflare R2（免费 10GB） |
| PWA | 支持离线编辑 |
| **月成本** | **约 ¥0-50** |

---

### 阶段 3：规模期（10 万 → 50 万用户）

**全栈分离 + 微服务**

| 组件 | 方案 | 月成本 |
|------|------|--------|
| CDN | Cloudflare Pro | ¥160 |
| 对象存储 | R2 (100GB+) | ¥40 |
| 数据库 | Supabase Pro | ¥180 |
| API | Vercel Pro | ¥160 |
| AI 推理 | API 调用 | ¥500-2000 |
| **合计** | | **~¥1000-2500** |

---

### 阶段 4：百万用户（50 万 → 100 万用户）

**企业级 + 全球化部署**

| 组件 | 方案 | 月成本 |
|------|------|--------|
| CDN + 安全 | Cloudflare Enterprise | ¥2000-5000 |
| 云服务器 | AWS/GCP 10-20 台 | ¥5000-15000 |
| 数据库 | PostgreSQL 集群 | ¥2000-5000 |
| AI 推理 | GPU 实例 | ¥5000-20000 |
| **合计** | | **~¥17000-49000** |

## 三、变现模型

| 方式 | 定价 | 月收入（百万用户） |
|------|------|------------------|

| **总计** | | **~¥110-120 万/月** |

## 四、增长路径

| 阶段 | 关键目标 | 手段 |
|------|---------|------|
| 0-1 万 | 验证产品价值 | 社区种草、口碑传播 |
| 1-5 万 | 获客引擎 | 社交媒体分享裂变 |
| 5-20 万 | 留存率提升 | AI 辅助降低门槛 |
| 20-50 万 | 付费转化 | Pro 功能上线 |
| 50-100 万 | 全球化 | 多语言、创作者激励 |

---

## 五、核心原则

> **不要为百万用户过早过度工程化。**

每一阶段只做该阶段需要的事。

- 阶段 1（现在）：钢琴卷帘好用、播放流畅、分享方便
- 等有 10000 个用户说"这个真好用"，再进化到阶段 2
